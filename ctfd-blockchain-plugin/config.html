Blockchain Challenge Plugin

Global Settings

These settings apply to all blockchain challenges unless overridden per-challenge.

RPC URL env var
BLOCKCHAIN_RPC_URL - set in your server environment to point to your EVM node (e.g. http://host.docker.internal:8545 for Anvil). Each challenge can also override this in its own settings.
Current value
{{ config.get("BLOCKCHAIN_RPC_URL") or "not set, falling back to http://host.docker.internal:8545" }}
Required Proxy Contract Interface

Your proxy contract must expose these two functions for the plugin to work: 

/// Deploy a fresh vulnerable instance for msg.sender.
/// Must emit ChallengeDeployed.
function deployChallenge(uint256 challengeIndex)
    external returns (address instance);

/// On-chain completion check - called by CTFd backend via eth_call.
/// Returns true once the player has successfully exploited their instance.
function isCompleted(address user, uint256 challengeIndex)
    external view returns (bool);

event ChallengeDeployed(
    address indexed user,
    uint256 challengeIndex,
    address instance
);

Plugin API Endpoints
MethodPathDescription
GET /api/v1/blockchain/challenge/<id>/info Returns chain_id, proxy address, index, ABI, source code
GET /api/v1/blockchain/challenge/<id>/deployment Player's current deployed contract address
POST /api/v1/blockchain/challenge/<id>/deployment Records a newly deployed instance address
POST /api/v1/blockchain/challenge/<id>/verify Triggers on-chain verification and awards CTFd points
